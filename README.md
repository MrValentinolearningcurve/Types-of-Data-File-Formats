# Types-of-Data-File-Formats
Understanding Data Formats and Exploring Structured, Semi-Structured, and Unstructured Data
Types of Data

1. Structured Data

2. Semi-Structured Data

3. Unstructured Data

4. Structured Data

## Structured Data

ข้อมูลที่มักเก็บอยู่ในรูปแบบโครงสร้างหรือตาราง (tabular format) แถวและคอลัมน์มีความเกี่ยวข้องกัน เช่น Excel, TSV, CSV

TSV ย่อมาจาก Tab-Separated Values

## Semi-Structured Data

ข้อมูลแบบมีโครงสร้างข้อมูลที่มีความยืดหยุ่น สามารถขยายโครงสร้างได้ในอนาคต อยู่ระหว่างข้อมูลแบบมีโครงสร้างและไม่มีโครงสร้าง เช่น JSON, Email, XML

## Unstructured Data

ข้อมูลที่ไม่มีโครงสร้าง ไม่มีการกำหนดโครงสร้างล่วงหน้าและไม่มี Data Model เช่น Text, numbers, audio, video, image, social media posts

## File Formats

- CSV ย่อมาจาก Comma-Separated Values คือ Text file ที่ไว้เก็บข้อมูลในรูปแบบของตารางคั่นด้วย Comma(,) หรือ Pipe (|)

- JSON เป็นรูปแบบข้อมูลจัดเรียงและจัดอยู่ในรูปแบบ Key-Values ที่ง่ายต่อการอ่านและเขียนสำหรับมนุษย์และคอมพิวเตอร์

- Parquet เป็นรูปแบบการจัดเก็บข้อมูลแบบไฟล์ (file format) ที่ออกแบบมาเพื่อการวิเคราะห์ข้อมูลขลาดกลาง (Big Data Analytics) โดยเฉพาะ รูปแบบ Parquet สามารถจัดเก็บข้อมูลที่มีโครงสร้างและประเภทข้อมูลต่าง ๆ ได้อย่างมีประสิทธิภาพ ซึ่งมีเป้าหมายในการลดขนาดข้อมูลที่จัดเก็บและเพิ่มประสิทธิภาพในการเรียกใช้ข้อมูล

รูปแบบ Parquet ใช้หลักการเก็บข้อมูลแบบคอลัมน์ (columnar storage) ซึ่งแตกต่างจากรูปแบบเก็บข้อมูลแบบแถว (row-based storage) ทั่วไป การจัดเก็บแบบคอลัมน์ช่วยลดการทำซ้ำของค่าในคอลัมน์เดียวกัน

> Parquet ช่วยให้การทำ Aggregation ได้เร็วเพราะดึงข้อมูลเฉพาะ column นั้นๆ แต่การดึงข้อมูลแบบ row จะดึงข้อมูลมาทั้งหมดแล้วค่อย filter ทีหลัง

> ในโลกของ Data จะแบ่งเป็น 2 ขั้วคือ
> 1.โลกของ Operation นิยมเก็บไฟล์ในรูปแบบ row storage 2.โลกของ Analytical ซึ่งจะนิยมเก็บไฟล์ในรูปแบบ columnar storage มากกว่า

- Binary format คือรูปแบบการเก็บข้อมูลที่ใช้รหัสข้อมูลในรูปแบบที่เข้าใจยากสำหรับมนุษย์ ซึ่งมักจะเป็นรหัสที่ใช้เลขฐานสอง (binary code) เพื่อแทนค่าข้อมูลต่าง ๆ ภายในไฟล์หรือโครงสร้างข้อมูล รูปแบบไบนารีใช้บิต (bit) เป็นหน่วยพื้นฐานในการจัดเก็บข้อมูล โดยทุกๆ บิตจะสามารถมีค่าเป็น 0 หรือ 1 เท่านั้น

  การใช้รูปแบบไบนารีสามารถช่วยลดขนาดของข้อมูลและเพิ่มประสิทธิภาพในการจัดเก็บและส่งข้อมูล ไบนารียังถูกนำมาใช้ในการเก็บข้อมูลที่ทำงานอยู่ในระดับฮาร์ดแวร์ เช่น การ   เก็บข้อมูลในหน่วยความจำแบบแรม (RAM) หรือการเขียนข้อมูลลงในอุปกรณ์เก็บข้อมูล

## Binary Format Over JSON in Data Product

ไฟล์ JSON นั้นอ่านและเขียนง่าย แต่มันไม่มีการกำหนดประเภทข้อมูลแบบเจาะจง

การไม่กำหนดประเภทข้อมูลใน Binary จะทำให้เกิดปัญหาหาก microservice ที่ใช้พัฒนาใช้กันคนละภาษา เนื่องจาก Data Type ของแต่ละภาษาก็ไม่เหมือนกันจึงต้องเจาะจงให้ชัดว่าเป็นข้อมูลประเภทไหน

ข้อมูล Binary Format มีประสิทธิภาพมากกว่า JSON เพราะไม่ต้องเผื่อ memory ว่าข้อมูลมี Data Type แบบไหน

ข้อมูลที่เก็บในรูปแบบ JSON หรือ CSV มี "-", "-" เพื่อให้มนุษย์อ่านได้ง่านแต่เปลืองพื้นที่ในการจัดเก็บหากไฟล์ที่ต้องการเก็บมีขนาดใหญ่มากๆ
